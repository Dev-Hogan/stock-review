## 目标

- 能够获取 A 股股票信息，历史信息，获取分钟级数据等
- 在web页面通过自然语言描述，大模型自动生成筛选因子代码
- 能够根据给定的模式或行为，筛选出符合条件的股票
- 提供可视化配置界面和 HTML 报告输出

## 技术栈

- **语言**: Python 3.11+
- **数据获取**: akshare
- **数据存储**: SQLite
- **大模型**: 本地部署 (通过 Ollama 调用)
- **CLI 框架**: Click 或 Typer
- **UI 配置**: Gradio
- **依赖管理**: Poetry
- **并发处理**: asyncio + aiohttp

## 功能模块

### 数据管理

- [ ] 获取 A 股股票基本信息 (名称、代码、行业、市值等)
- [ ] 获取日 K 线历史数据 (近 1 年，约 250 个交易日)
- [ ] 获取分钟级数据 (1 分钟/5 分钟/15 分钟/30 分钟/60 分钟)
- [ ] 数据自动更新 (盘后批量更新)
- [ ] SQLite 数据存储与缓存
- [ ] 数据获取失败自动重试 3 次，失败后跳过并记录日志

### 因子系统

- [ ] 自然语言转因子代码 (通过大模型)
- [ ] 支持三种输入方式:
  - 命令行参数：`stock filter --pattern "放量上涨且突破 20 日均线"`
  - 交互式对话：CLI 对话模式逐步询问条件
  - 配置文件：YAML/JSON 格式策略定义
- [ ] 生成的因子代码需用户审核确认后才执行
- [ ] 因子代码安全校验 (防止危险操作)
- [ ] 常用因子模板库 (技术指标、量价关系、形态识别等)

### 筛选引擎

- [ ] 支持全 A 股筛选 (5000+ 股票)
- [ ] 支持自定义股票池
- [ ] 异步并发处理 (提升筛选速度)
- [ ] 多条件组合筛选
- [ ] 结果排序 (按匹配度、涨跌幅、成交量等)

### 结果输出

- [ ] JSON 格式输出
- [ ] Markdown 表格输出
- [ ] HTML 报告 (含基本图表)
- [ ] 筛选结果保存到历史记录

### 配置管理

- [ ] Gradio Web 配置界面
- [ ] 大模型配置 (Ollama 地址、模型名称)
- [ ] 数据源配置
- [ ] 筛选参数预设
- [ ] 配置文件持久化

### CLI 命令

- [ ] `stock init` - 初始化配置
- [ ] `stock fetch` - 获取/更新数据
- [ ] `stock filter` - 执行筛选
- [ ] `stock report` - 生成报告
- [ ] `stock config` - 打开配置界面
- [ ] `stock factors` - 管理因子模板

## 非功能性需求

- **性能**: 全市场 5000+ 股票筛选应在 5 分钟内完成
- **可靠性**: 数据获取失败自动重试 3 次
- **可扩展**: 易于添加新的因子模板和数据源
- **安全性**: 生成的代码需安全校验，禁止危险操作
